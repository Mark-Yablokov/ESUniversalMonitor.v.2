# ESUniversalMonitor – Контекст проекта

**Версия:** 0.2.0 (исправленная)
**Дата обновления:** 2026-04-23

## 1. Назначение системы

**ESUniversalMonitor** — это десктопное приложение на базе фреймворка **PyQt5**, предназначенное для автоматизации процесса поверки (калибровки) электросчётчиков и промышленных измерительных приборов.  
Основные функции:

- Подключение к измерительным приборам (мультиметры, счётчики) и источникам испытательных сигналов (генераторы, высоковольтные источники).
- Ручная установка параметров тестового сигнала и выполнение единичных измерений.
- Автоматическое прохождение тестовых точек по заданной методике с проверкой допусков.
- Визуализация live-графиков измеряемых величин (Dashboard).
- Сохранение истории измерений в CSV и загрузка методик из JSON.

## 3. Поддерживаемое оборудование

- **Мультиметры:** Rigol DM3068 (подключение LAN / PyVISA)
- **Универсальные устройства:** поддержка Modbus RTU/TCP (собственный драйвер `modBus.py`)
- **Генераторы / эталоны:** MTE / EMH PTSx.xC (RS232)
- **Высоковольтные источники:** Mantigora HT / HP (RS232)

## 4. Основные возможности приложения

- **Многопоточность:** одновременный опрос нескольких панелей без блокировки GUI.
- **Dashboard:** построение графиков измеряемых величин в реальном времени с возможностью загрузки истории из CSV.
- **Ручной режим:** установка напряжения, тока, частоты и фазы; выполнение одиночных измерений; сравнение с заданными допусками.
- **Автоматический режим:** последовательная отработка тестовых точек по загруженной методике (JSON), проверка допусков, формирование отчёта.
- **Расчёт погрешностей:** абсолютной, относительной, приведённой.

## 5. История изменений и текущее состояние (последние правки)

**Дата:** 2026-04-23

### Выполненные исправления и улучшения

1. **Исправление сигнатуры метода `connect` в базовом генераторе**
   - Файл: `generators/base_generator.py`
   - Проблема: метод `connect()` был объявлен без параметров, но наследники (`PTSGenerator`, `MantigoraGenerator`) принимали `device_panels`.
   - Решение: добавлен опциональный параметр `device_panels` в абстрактный метод.

2. **Передача параметра `hp_mode` в драйвер Mantigora**
   - Файл: `panels/mantigora_panel.py`
   - Проблема: в UI есть выбор серии HT/HP, но этот флаг не передавался в конструктор `MantigoraDriver`.
   - Решение: добавлено преобразование серии в булево значение `hp_mode` и его передача при создании драйвера.

3. **Устранение дублирования датаклассов**
   - Создан новый модуль `core/measurement_types.py`, содержащий общие структуры:
     - `ToleranceSpec`
     - `ParameterLink`
     - `TestPoint`
   - Файлы `tabs/manual_generation_tab.py` и `tabs/auto_test_tab.py` переписаны с импортом из `core`.
   - Удалены локальные определения классов и «запасной» импорт.

4. **Создание пакета `core`**
   - Добавлен файл `core/__init__.py` для явного экспорта основных типов.

5. **Формирование списка зависимостей**
   - Добавлен файл `requirements.txt` с библиотеками:
     - `PyQt5`, `pyqtgraph`, `numpy`
     - `pyvisa`, `pyvisa-py`
     - `pyserial`

### Текущее состояние

- Проект готов к дальнейшей разработке и тестированию.
